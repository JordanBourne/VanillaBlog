Why Use Vanilla Javascript

By Jordan Bourne / 2018-06-28

Let's just start by saying there is absolutely a point in developing any large-scale web application that it probably would have been better to start off with a framework or library. For those who are still early on the path of learning, or perhaps you already have a bit of experience and you are just starting a new personal project, Vanilla JavaScript can be a great choice.

The first thing we need to establish is that, by definition, anything you can do with the a framework or library can be done with Vanilla JavaScript, and it's done more simply than you would expect. What is it you're trying to do, create a single page application with reuseable components? We've got you covered. Use the virtual DOM to increase the ease of which we can update the view? Check.

Why not just use an established framework already if our plan is to reimplement the features of one anyway? There are two main reasons as I see it:

Learning how everything works under the hood.

When you embark on your journey to implement many of the common feature found in the frameworks of vogue, you learn much more about the way things work. The cross-over of skill between each framework and library can be limited, whereas understanding the principles of JavaScript will be of great assistance when transitioning your skills in the future.

Focus on solving problems, not finding the tools.

What I love about programming is how such complex problems or applications can be broken down and solved using such simple tools. Using just Vanilla JavaScript allows you to just jump straight into building the thing you want to build. There's no debate or worry over picking the right framework, and you don't have to worry about configuring your bundlers or compilers. Just include it on your page and it will just work.
